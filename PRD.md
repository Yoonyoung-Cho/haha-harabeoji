# 우리들의 즐거운 시간 (우즐시) — 제품 PRD(기획)

> **슬로건**: “매일 아침, 당신의 입가에 피어나는 작은 미소”

## 1. 제품 개요

- **서비스 명칭**: 우리들의 즐거운 시간 (우즐시)
- **플랫폼**: 웹 앱(모바일 브라우저 중심, 추후 PWA 가능)
- **목표 사용자**: 70~90대 어르신(시력이 약하고 스마트폰 사용이 서툰 분 포함)
- **핵심 가치**: 쉽고 안전하게 “웃을 거리”와 “따뜻한 글”을 보고, 가볍게 반응하고, 누군가에게 공유하며 **외로움을 줄이는 경험**

## 2. 문제 정의

- **접근성 문제**
  - 어르신이 스스로 유머/좋은 글/건강 상식을 찾기 어렵고, 기존 서비스 UI가 복잡하다.
- **관계 축소와 외로움**
  - 나이가 들수록 주변 지인이 줄고, 카톡으로 주고받던 자료 공유가 감소하며 **소통이 줄어든다**.
- **콘텐츠 안전성**
  - 정치/선정성/혐오/허위 건강정보 등은 고령층에게 불쾌·불안·분열을 유발할 수 있어 **강한 필터링이 필요**하다.

## 3. 목표 및 성공 지표

- **제품 목표**
  - 하루에 1~3분이라도 “부담 없이 웃고/고개 끄덕일 수 있는” 습관을 만들도록 돕는다.
  - 글을 저장(좋아요)하고 공유하며, **소통의 계기**를 만든다.

- **정량 지표(예시)**
  - 7일 유지율: 35%+
  - 평균 1회 접속 체류시간: 3분+
  - 공유하기 클릭 비율: 15%+
  - 좋아요(저장) 클릭 비율: 25%+

## 4. 사용자 페르소나(핵심)

- **페르소나 A: 78세 할아버지**
  - 스마트폰은 “카톡/전화/사진” 정도만 사용.
  - 친구가 줄고 카톡방이 조용해져서 **심심함·외로움**을 자주 느낌.
- **페르소나 B: 82세 할머니**
  - 작은 글씨/복잡한 메뉴에 피로감을 느낌.
  - “좋은 글/건강 이야기”를 가족에게 보내며 대화 소재를 만들고 싶음.
- **페르소나 C: 40~50대 자녀**
  - 부모님이 혼자 있는 시간이 걱정됨.
  - 정치/선정성 없는 안전한 콘텐츠로 부모님이 편하게 즐기길 바람.
- **페르소나 D: 복지관 강사/사회복지사**
  - 수업에서 쓰기 쉬운 단순한 앱을 선호(설치/사용법 간단).

## 5. UX/디자인 원칙

- **큰 글씨, 큰 터치 영역**
  - 본문 최소 24px 이상, 버튼 터치 영역 최소 48dp 이상 권장.
- **한 화면에 한 가지 행동**
  - 스크롤로 보기 → (선택) 평가 → (선택) 저장 → (선택) 공유
- **아이콘 + 짧은 텍스트**
  - 텍스트만 나열하지 않고, 직관적인 아이콘을 함께 사용.
- **눈이 편한 미색 배경 + 고대비**
  - 배경은 따뜻한 미색 톤, 글자는 진한 색으로 대비 확보.

## 6. 핵심 기능(MVP)

### 6.1 스크롤 피드(유머/건강/지혜) + 뷰 모드 토글

- **설명**: 카드형 콘텐츠를 세로 스크롤로 넘겨서 본다.
- **카테고리**
  - 유머 / 건강 / 지혜 (전체 보기 포함)
- **뷰 모드 전환**
  - **카드형** (기본): 제목+본문이 카드로 펼쳐진 스크롤 피드
  - **게시판형**: 제목 목록만 보이고, 클릭하면 상세 글이 보이는 전통 게시판 형태
  - 헤더 우측의 토글 버튼으로 전환

### 6.2 좋아요(저장) + “내가 좋아요 누른 게시글” 모아보기

- **의도**: “나중에 다시 보고 싶은 글”을 저장하는 기능.
- **UX**
  - 카드 우측 상단 하트로 저장/해제.
  - 상단에 “내가 좋아요 누른 게시글” 진입(마이페이지처럼 토글) 제공.

### 6.3 평가(따봉 👍/👎) + 다음 카드로 이동

- **의도**: 댓글 없이도 “재밌다/별로다”를 매우 간단히 표현.
- **UX**
  - 👍/👎 중 하나만 선택.
  - 누르면 자동으로 다음 카드로 부드럽게 이동.

### 6.4 공유하기(카카오톡 등)

- **의도**: 가족/지인과의 소통 계기 제공.
- **UX**
  - “공유하기” 버튼으로 즉시 공유(가능하면 시스템 공유 시트).
  - 카드 우측 상단에 공유 횟수(표시용)를 함께 보여줌.

## 7. 콘텐츠 전략 및 금지 정책

### 7.1 추천/선호되는 콘텐츠

- **유머/재미**: 짧은 생활 유머, 옛날 이야기, 가벼운 상황극
- **건강**: 생활 습관, 스트레칭, 식습관 팁(의료 진단/처방 아님 명시)
- **지혜/좋은 글**: 위로, 공감, 가족/관계, 마음 다스리기

### 7.2 절대 금지(필수)

- **정치 관련**: 정당/정치인/선거/시사 풍자 포함 “일절 금지”
- **성적 콘텐츠**: 노골적/은유적 성표현, 선정성, 음담패설 금지
- **혐오/차별/폭력/불안 조장**: 비하/조롱/공포 과도 유발 금지
- **허위 건강정보/과장 광고**: 검증되지 않은 민간요법, 기적의 약 등 금지

### 7.3 저작권/재가공 원칙

- 외부 커뮤니티 글은 **원문 길이·어휘를 유지하는 것을 기본**으로 한다.
- 다만, 정치/광고/성인/혐오/허위 건강정보 등 **정책 위반 요소가 포함된 글은 통째로 배제(표시하지 않음)** 한다.
- 출처(카페명/게시판명/작성자명 등)는 **카드 안에 “출처: ○○카페 ○○게시판” 형식으로 명시**한다.
- 이미지/영상은 라이선스가 명확한 소스만 사용하며, 필요 시 썸네일 수준으로만 활용한다.

### 7.4 수집 및 표시 방식 (원문 그대로 + 출처 표시)

- **수집**
  - 다음카페/네이버카페/기타 커뮤니티 등에서 **어르신들이 많이 찾는 게시판(유머/좋은글/건강 등)** 의 글을 수집한다.
  - 길이나 어휘를 인위적으로 줄이거나 풀지 않는다.  
    (예: 문장 길이 유지, 표현도 원문 그대로 유지)
- **필터링**
  - 수집 단계에서 **정치/광고/성인/혐오/허위 건강정보**가 감지되면 해당 글은 전체를 제외한다.
  - 필터링은 키워드·규칙 기반 + 운영자 검토(초기)를 병행할 수 있다.
- **표시**
  - 우즐시는 가능한 한 **원문 그대로**를 카드에 노출한다.
  - 카드 안에 항상 **“출처: ○○카페 ○○게시판”과 원문 링크**를 함께 표기한다.
  - 우즐시에서의 2차 가공은 최소화하고, 필요 시 **카테고리 태그/요약 한 줄 정도만 덧붙이는 수준**으로 제한한다.

## 8. 개인정보/데이터 원칙

- **최소 수집**: 이름/전화번호 등 민감정보는 기본적으로 수집하지 않는 방향
- **저장 목적 명확화**: 좋아요/평가/공유 등은 UX 개선 및 개인 편의 목적
- **로컬 우선**: MVP는 서버 없이 단말(localStorage) 저장을 기본으로 시작 가능

## 9. 운영/모더레이션(초기 원칙)

- 초기에는 운영자 큐레이션 중심으로 운영(콘텐츠 선별/수정/배포)
- 신고 기능은 추후 확장(필요 시)하되, 최소한의 카테고리로 단순화

## 10. 롤아웃/성장 전략(초기)

- **클로즈드 베타**: 가족 단위, 복지관/경로당 소규모에서 사용성 피드백 수집
- **확산 채널**: 복지관 스마트폰 교육, 자녀 대상 “부모님께 설치” 캠페인

## 11. 로드맵(예시)

- **MVP**
  - 피드/카테고리, 좋아요 저장+모아보기, 👍/👎 평가+다음 카드 이동, 공유하기
- **차기**
  - TTS(읽어주기), 글자 크기 단계 옵션, 관리자 콘솔, 콘텐츠 추천 고도화

## 12. 기술 스택(권장)

- **저장**: MVP는 localStorage 우선 → 필요 시 서버/DB로 확장

# (1) 기능 명세

## 1.1 큰 글씨·큰 버튼 UI (접근성)

- **설명**: 모든 텍스트와 버튼을 어르신이 읽기·누르기 쉽게 크게 표시한다.
- **구현 포인트**
  - 전역 스타일: Tailwind로 `text-2xl`(24px) 이상 적용.
  - 버튼: `min-h-[56px] px-6 text-2xl` 이상, 아이콘은 `size-8` 이상.
  - 터치 영역: 클릭 가능 영역 최소 48×48px 이상 유지.
- **완료 조건**
  - [ ] 본문이 24px 미만인 화면이 없음.
  - [ ] 모든 버튼이 한 손가락으로 쉽게 눌리도록 크기가 충분함.

---

## 1.2 스크롤 유머 피드

- **설명**: 유머/건강/지혜 등 카테고리별 카드를 세로 스크롤로 넘기며 본다.
- **동작**
  - 세로 스크롤만으로 다음 카드로 이동 (스와이프 또는 스크롤).
  - 한 카드에 하나의 콘텐츠(텍스트 또는 이미지 한 장).
  - 카드 하단에 반응 버튼(웃었어요 / 그저 그래요 / 옛날 생각나요)과 공유하기 버튼.
- **구현 포인트**
  - 페이지: `/` 또는 `/feed` — 피드 메인.
  - 카드 컴포넌트: `FeedCard` — 제목(선택), 본문, 이미지(선택), 반응·공유 버튼.
  - 데이터: 초기에는 정적 JSON/배열로 제공해도 됨. 이후 API로 교체.
- **완료 조건**
  - [ ] 스크롤 시 카드가 순서대로 보임.
  - [ ] 카드 본문 글자 크기 24px 이상.
  - [ ] 반응 버튼 클릭 시 상태 반영(선택 표시 등).

---

## 1.3 카테고리 (유머 / 건강 / 지혜)

- **설명**: 피드 상단 또는 첫 진입 시 카테고리를 고를 수 있다.
- **카테고리 종류**
  - **유머**: 짧은 농담, 옛날 유머, 생활 유머.
  - **건강**: 생활 습관, 운동·식습관 팁 등 (의료 진단 아님 명시).
  - **지혜**: 좋은 글, 인생 이야기, 가족·관계에 대한 짧은 글.
- **동작**
  - “전체” 또는 특정 카테고리 선택 시 해당 카드만 필터링해 표시.
  - 선택한 카테고리는 URL 또는 로컬 스토리지에 저장해 다음 방문 시 유지 가능(선택).
- **구현 포인트**
  - `category` 값: `'all' | 'humor' | 'health' | 'wisdom'`.
  - 상단에 큰 탭 또는 버튼 4개: 전체 / 유머 / 건강 / 지혜.
- **완료 조건**
  - [ ] 카테고리 선택 시 해당하는 카드만 노출.
  - [ ] 선택 상태가 시각적으로 명확함.

---

## 1.4 카카오톡으로 공유하기

- **설명**: 현재 보고 있는 카드 내용을 카카오톡으로 공유한다.
- **동작**
  - 카드 하단 “공유하기” 버튼 클릭 → 카카오톡 공유(또는 공유 메뉴) 실행.
  - 공유 시 제목 + 링크 + 요약 텍스트(선택) 전달.  
    (웹이므로 실제 전달은 “현재 페이지 링크 + 제목” 위주로 구현.)
- **구현 포인트**
  - 웹: **Kakao JavaScript SDK** 또는 **Web Share API** 사용.
  - `navigator.share()` 지원 시: 제목, 텍스트, url 넘겨서 “카카오톡”이 선택지에 포함되도록.
  - 미지원 시: `https://story.kakao.com/share` 또는 `https://developers.kakao.com` 공유 URL 스킴 활용.
  - 버튼: Lucide `Share2` 아이콘 + “공유하기” 텍스트, `text-2xl` 이상.
- **완료 조건**
  - [ ] “공유하기” 버튼 클릭 시 공유 동작(Web Share 또는 Kakao) 실행.
  - [ ] 공유 시 보여줄 문구(예: “이거 보고 웃으세요”) 고정 또는 간단 설정 가능.

## 1.5 반응 버튼 (웃었어요 / 그저 그래요 / 옛날 생각나요)

- **설명**: 댓글 대신 한 번에 누를 수 있는 반응 3종.
- **동작**
  - 한 카드당 하나만 선택 가능(선택 시 이전 선택 해제).
  - 선택 시 해당 버튼 시각적 강조(배경색/테두리 변경).
  - 초기 버전: 선택 상태만 로컬(브라우저)에 저장해도 됨.  
    이후 서버에 “이 카드에 웃었어요 312명”처럼 집계 표시 가능.
- **구현 포인트**
  - 반응 타입: `'laugh' | 'soso' | 'nostalgia'`.
  - Lucide: 웃었어요 `Laugh`, 그저 그래요 `Meh`, 옛날 생각나요 `Heart` 등.
  - 상태: `React.useState` 또는 URL 쿼리로 “카드 id + 선택한 반응” 저장.
- **완료 조건**
  - [ ] 세 버튼 모두 24px 이상 글자·아이콘.
  - [ ] 클릭 시 선택 상태 토글/변경 및 시각적 피드백.

# (3) 화면 설계 (와이어프레임)

## 3.2 피드 메인 (`/`)

- **역할**: 카테고리 선택 + 카드 리스트 스크롤 + 반응·공유.

**와이어프레임 (모바일 세로)**

```
+------------------------------------------+
|  [배경: 미색 #F5F0E8]                    |
|  --------------------------------------- |
|  할아버지 할머니를 위한 유머집  (로고/타이틀, text-4xl)
|  --------------------------------------- |
|  [전체] [유머] [건강] [지혜]   (큰 탭 버튼, 24px+, 미색 위에 진한 테두리)
|  --------------------------------------- |
|                                          |
|  +------------------------------------+  |
|  |  [카드 1]                           |  |
|  |  제목(선택) text-3xl                |  |
|  |  본문 본문 본문... text-2xl~3xl     |  |
|  |  (이미지 있으면 표시)               |  |
|  |  --------------------------------- |  |
|  |  [웃었어요] [그저그래요] [옛날생각]  |  |
|  |  [공유하기] (Share2 아이콘 + 글자)   |  |
|  +------------------------------------+  |
|                                          |
|  +------------------------------------+  |
|  |  [카드 2] ...                       |  |
|  +------------------------------------+  |
|                                          |
+------------------------------------------+
```

- **구성 요소**
  - **헤더**: 앱 이름만 단순히 표시 (네비 없이 단순).
  - **카테고리**: 4개 버튼(전체/유머/건강/지혜), 선택된 것은 배경색으로 강조.
  - **카드**: `FeedCard` — 제목(있을 때), 본문, 이미지(있을 때), 반응 3버튼 + 공유하기 버튼.
  - **푸터**: 첫 버전에서는 생략 가능.

# (4) 개발 우선순위

아래 순서대로 구현하면, “볼 수 있는 피드 → 반응 → 공유”까지 최소 경로로 완성할 수 있습니다.

| 순위 | 작업 | 상세 | 예상 결과 |
|------|------|------|-----------|
| **1** | 프로젝트 셋업 | Next.js(App Router), Tailwind, Lucide-react 설치 | `npm run dev`로 빈 페이지 실행 가능 |
| **2** | 전역 디자인 | layout에서 배경 미색, 기본 `text-2xl`, 고대비 색상 적용 | 모든 페이지가 미색·큰 글자 기준 적용 |
| **3** | 데이터 타입 + 목업 데이터 | `ContentCard` 타입 정의, `data/feed.json` 또는 ts 배열로 카드 5~10개 준비 | 타입과 샘플 데이터 존재 |
| **4** | 피드 메인 페이지 | `/`에서 카드 배열을 map으로 렌더, 카드별 제목·본문·이미지 표시 | 스크롤되는 카드 리스트만 보임 |
| **5** | 카드 컴포넌트 정리 | `FeedCard`로 제목/본문/이미지 분리, 버튼 영역 자리만 확보 | 카드 UI가 컴포넌트로 정리됨 |
| **6** | 카테고리 필터 | CategoryTabs + 상태로 `all`/humor/health/wisdom 필터링 | 탭 선택 시 해당 카테고리만 표시 |
| **7** | 반응 버튼 | 웃었어요/그저 그래요/옛날 생각나요 클릭 시 선택 상태 저장(useState 또는 localStorage) | 반응 선택 시 시각적 반영 |
| **8** | 공유하기 | Web Share API 또는 Kakao 공유 연동, “공유하기” 버튼에 연결 | 버튼 클릭 시 공유 동작 |
| **9** | 반응 집계(선택) | 서버/API 없이 “이 카드에 N명이 웃었어요” 같은 고정 문구 표시 | UX 보강 |
| **10** | 접근성 점검 | 버튼 최소 크기, 포커스 순서, 색 대비 재확인 | 어르신 사용 시나리오에 맞게 조정 |
